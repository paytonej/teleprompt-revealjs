Slide 1
This is the notes content for the slide. It will be exported to a text
file when the presentation is rendered, and can be used as a
teleprompter script.
----------------------------------------
Slide 2
The notes sections are separated by slide in the output text file. So,
for example, the notes content for this slide will be in a section of
the output text file that is separate from the notes content for the
previous and next slides. This allows you to easily organize your
teleprompter script by slide.
----------------------------------------
Slide 3
We can adjust the background color of the presentation to make it
suitable for use as a background for a smart-board or teleprompter. For
example, if we are recording in front of a green screen for chroma
keying, we can set the background color to green so that the
presentation will be invisible when used as the background for the
smart-board, and our text and content will appear on top of whatever we
want to use for the background.
----------------------------------------
Slide 4
The navigation controls are forced to the background color by default so
that they will not be visible when the presentation is used as the
background for the smart-board. However, if you still want to see
navigation controls, you can set their color using the
--teleprompt-nav-color CSS variable in the include-in-header field in
the YAML front matter. For example, you can set the navigation controls
to yellow by adding code to the include-in-header.
----------------------------------------
Slide 5
If you’re standing on a different side of the slide, or moving around a
lot, you might want to adjust the content alignment and width. You can
do this using CSS variables on a per-slide basis. For example, you can
make the content column right-aligned and 80% of the slide width by
adding the teleprompt-right class and setting the --teleprompt-col-width
variable in the markdown for the slide.
----------------------------------------
Slide 6
Like in LaTeX, the font size of each content div can be set using LaTeX
font size class names. This allows you to easily adjust the font size of
your content to make it more readable or to fit more content on the
slide as needed. For example, you can use the font-large class to make
the text a large size, or the font-small class to make it a smaller
size.
----------------------------------------
Slide 7
The “bang box” is a custom class that can be added to any content div to
make it stand out with a red border and background. This is useful for
drawing attention to a key point or conclusion on the slide. For
example, you can use the bang-box class to highlight an important
message or takeaway that you want the audience to remember.
----------------------------------------
Slide 8
The source-citation class can be added to any content div to format it
as a source citation, with a tiny font size, gray color, and placement
at the bottom of the slide. This is useful for giving credit to sources
or providing additional information without distracting from the main
content of the slide. For example, you can use the source-citation class
to cite a source for an image or a piece of information that you are
including on the slide.
----------------------------------------
Slide 9
The default text color for this theme is white (with red links), but you
can change the text color of any content div using the color CSS
property. For example, you can set the text color to blue by adding a
style attribute with color: blue; to the content div. The choice of text
color will depend on the background color you are using in your video,
as you want to make sure the text is readable against the background.
----------------------------------------
Slide 10
The previous slide showed the default code block appearance, which has a
dark background and light text. However, there is also an option for a
light background and dark text, which might be preferable if the code
block needs to stand out against a dark background. To use this style,
simply add the code-light class to the slide block, as shown in the
markdown code. This will change the appearance of all code blocks on
that slide to have a light background and dark text.
----------------------------------------
Slide 11
The extensions from quarto-revealjs-coeos package are loaded
automatically, enabling tabsets and per-line code annotation
capabilities in your presentation. You can use these features to
organize your content into tabs or to annotate specific lines of code in
your code blocks, which can be very helpful for explaining complex code
or concepts in a clear and organized way. For example, you can create a
tabset to show different pieces of content in the same area of the
slide, or you can use per-line code annotation to highlight specific
lines of code and provide explanations for each line.
